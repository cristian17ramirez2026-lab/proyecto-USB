Wireframes del Sistema SIGAE

Sistema de Gestión de Inventario y Control de Activos Empresariales — 11 Vistas Completas

Wireframe 1: Pantalla de Login

Notas: Diseño split-screen que separa la funcionalidad (izquierda) de la identidad visual (derecha). El degradado rojo refuerza la marca institucional. Se incluyen credenciales demo para facilitar pruebas. Los campos usan labels en mayúsculas para jerarquía visual clara. El botón rojo oscuro (#8B0000) genera contraste y destaca la acción principal.

Wireframe 2: Dashboard + Layout Principal

Notas: Layout con sidebar fijo de 200px que agrupa la navegación por secciones lógicas (Principal, Inventario, Organización, Análisis, Administración). El Dashboard muestra 8 KPIs en grid 4×2 con colores semánticos: rojo para totales, verde para disponibilidad, naranja para asignaciones, azul para información. La tabla de actividad reciente proporciona contexto inmediato de las últimas operaciones del sistema.

Wireframe 3: Lista de Activos

Notas: La tabla de activos incluye búsqueda en tiempo real y filtro por estado. Los badges de estado usan colores semánticos (verde=disponible, amarillo=asignado, rojo=dañado) para identificación rápida. Los botones de acción principales (Nuevo Activo y CSV) están posicionados en la esquina superior derecha siguiendo convenciones de UI.

Wireframe 4: Formulario de Activo

Notas: Formulario organizado en grid de 3 filas con campos agrupados lógicamente. Los dropdowns de Sede y Departamento se cargan dinámicamente. La descripción ocupa el ancho completo para permitir texto extenso. Los botones de acción siguen el patrón Cancelar (gris) + Guardar (azul primario) alineados a la derecha.

Wireframe 5: Asignaciones

Notas: La vista combina formulario de nueva asignación (parte superior) con historial (tabla inferior). Solo se muestran activos con estado "Disponible" en el dropdown. El botón "Devolver" solo aparece en asignaciones activas. Los badges amarillo (Activa) y verde (Devuelta) permiten identificar rápidamente el estado de cada asignación.

Wireframe 6: Empleados

Notas: La tabla muestra nombre y email en la misma celda para optimizar espacio. El badge de activos asignados permite ver rápidamente la carga de cada empleado. Los empleados dados de baja se muestran con opacity 50% y badge rojo, con opción de "Reactivar" en lugar de "Baja". Las acciones contextuales cambian según el estado del empleado.

Wireframe 7: Sedes (Solo Admin)

Notas: Vista exclusiva para administradores (badge "Solo Admin"). Los badges azul (departamentos) y morado (activos) muestran conteos asociados a cada sede. El formulario inline aparece al hacer clic en "Nueva Sede" o "Editar", evitando navegación adicional. Se incluyen campos esenciales: nombre, ciudad, dirección y teléfono.

Wireframe 8: Departamentos (Solo Admin)

Notas: Vista exclusiva para administradores. El dropdown de Sede permite asociar departamentos a sedes existentes. Los badges azul (área) y morado (empleados) proporcionan contexto visual rápido. El formulario inline sigue el mismo patrón de Sedes para consistencia. Se muestran 3 departamentos de ejemplo con diferentes sedes para ilustrar la relación.

Wireframe 9: Reportes

Notas: La vista de reportes combina tarjetas resumen (KPIs) con tabs filtrables por estado. El tab activo (rojo) muestra los activos disponibles por defecto. La exportación CSV permite descargar los datos filtrados. Los colores de las tarjetas son semánticos: verde para valor, azul para totales, naranja para asignados, rojo para reparación.

Wireframe 10: Usuarios (Solo Admin)

Notas: Vista exclusiva para administradores. El formulario de creación está integrado en la parte superior. Los roles se distinguen con badges de color: rojo para ADMIN y azul para OPERADOR. La sede "Todas" indica acceso global para administradores, mientras que los operadores están limitados a su sede asignada. Se incluyen acciones de editar y eliminar por usuario.

Wireframe 11: Registro de Actividad (Solo Admin)

Notas: Vista exclusiva para administradores que registra todas las acciones del sistema. Los badges azules en la columna ACCIÓN categorizan visualmente cada tipo de evento (Asignación, Activo creado, Usuario creado, Devolución, Login). El botón "Limpiar log" tiene estilo outline (borde gris) para indicar que es una acción destructiva que requiere confirmación. El registro se ordena cronológicamente de más reciente a más antiguo.
